## What the Repo Charter Contains

The current charter at `charters/smoking-tigers-charter.md` (extracted from STE_Knowledge_Bible_v0.3.md) includes:

- **Governance Hierarchy:** Q1 Governance → CCA → Rev Token Purchase Agreement → Project Activation Agreement → Project-specific → Policies
- **Core Principles:** Contribution ≠ Employment, Stewardship ≠ Ownership, Projects as unit of action, Explicit IP, Bounded recovery, Right to exit
- **Decision Domains Table:** Creative Direction, Budget Allocation, Revenue Allocation Changes, Multiplier Framework, Scope Expansion, Pause/Termination, Charter Amendments — with defined Proposer/Decider/Veto roles
- **Voting Mechanics:** One participant one vote, >50% quorum, simple majority approval, Steward tie-breaker
- **Agreement Architecture:** Full stack from CCA down to IP License Agreements (Section 1.6)

## Gaps Summary

1. **Content Stewards role** is not defined in the repo charter
2. **Ad Hoc Working Groups** have no framework or formation rules
3. **Project Assembly** is referenced but not formally defined (composition, eligibility, formation)
4. **Approval threshold** should be explicitly stated as 51% rather than "simple majority"
5. The **NBA master services agreement analogy** that shapes the v2 draft is not reflected

## Recommendation

Ed should review the Notion Charter Agreement Draft document and decide whether to:

1. **Update the existing charter** to add the missing structural definitions (Content Stewards, Working Groups, Assembly composition)
2. **Replace the charter** with a v2 that formally adopts the NBA MSA model
3. **Keep both** — current charter as operational baseline, v2 as the target for formal adoption by Assembly vote

The core voting mechanics are already aligned. The main gaps are in **organizational structure definitions** (Assembly, Stewards, Working Groups) rather than decision-making rules.
